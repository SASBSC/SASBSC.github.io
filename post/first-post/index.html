Hugo + Github Pages + Github Actions 搭建个人博客

May 26, 2022 in 生产力, 个人博客

记录个人第一次博客搭建之路

需要用到的软件或工具

  • Hugo 静态网站生成器
  • Github Pages 网站部署
  • Github Actions 自动化网站部署
  • Gittalk 评论系统
  • Visual Studio Code (VS Code) 编辑器

总体思路

VS code 编辑文章，并使用本地预览。编辑好新文章后使用 git 推送到 GitHub 私有源码仓库 blog-source-code，此时触发 Github Actions，按照指定流程生成新的网页文件并存入公开页面仓库 username.github.io，也即 Github Pages 仓库。

安装Hugo

Hugo是一个用Go语言编写的静态网站生成器，非常适合搭建静态博客。类似的生成器还有Hexo、Jekyll等。参考网上的信息，考虑到搭建难易性及页面加载速度，最终选择Hugo。

Hugo 官方主页

下载对应系统版本的编译好的Hugo二进制文件。解压后放入指定安装目录，然后将该目录添加至系统环境变量（win+R - sysdm.cpl - 高级 - 环境变量 - 系统变量 - Path）

配置GitHub Pages 和 Github Actions

生成两个仓库

  • blog-source-code 私有（Private）
  • username.github.io 公开（Public）

生成 personal_token

由于源码和网页分开存放，生成的网页文件推送到 Github Pages 仓库需要相应权限，此处采用 personal_token 的方式。

依次点击 Github 头像 - Setting - Developer Settings - Personal access tokens - Generate new token，填入名字及备注（可选），并全选 repo 及 admin:repo_hook 选项，点击 Generate token 生成 token，复制保存该token。 只出现一次，务必先复制保存

配置源码仓库

将空的源码仓库克隆到本地

git clone git@github.com:username/blog-source-code.git

初始化 Hugo 站点，在指定的站点目录执行语句

hugo new site ./blog-source-code

安装主题

在 Hugo Themes 网站选择合适的主题，下文以 Tranquilpeak 为例。

将选中的主题克隆到本地源码文件夹内 themes 文件夹下

cd blog-source-code
cd themes
git clone https://github.com/kakawait/hugo-tranquilpeak-theme.git hugo-tranquilpeak-theme

将 /themes/tranquilpeak/exampleSite 下的 config.toml 复制过来覆盖 /blog-source-code 下的原文件，打开 config.toml 阅读并修改相应配置。

添加子模块

git submodule add https://github.com/kakawait/hugo-tranquilpeak-theme.git

推送到 Github 源码仓库

git add .
git commit -m "New site!"
git push -u origin master

源码仓库添加 personal_token

进入源码仓库（blog-source-code），依次点击 Setting - Secrets - Action - New Repository secret，填入上面生成的 token，命名为 ACTION_ACCESS_TOKEN。注意此处可随意命名，只需与后续 Github Actions 配置文件保持一致即可。

配置 Github Actions

进入源码仓库（blog-source-code），依次点击Actions - set up a workflow yourself， 复制修改下述代码，并保存为main.yml，commit即可。

name: GitHub Pages

on:
push:
    branches:
    - main

jobs:
deploy:
    runs-on: ubuntu-20.04
    steps:
    - name: Checkout repositories
        uses: actions/checkout@v3
        with:
        submodules: true  # Fetch Hugo themes (true OR recursive)
        fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod

    - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
        hugo-version: 'latest'
        extended: true

    - name: Build
        run: hugo

    - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
        personal_token: ${{ secrets.ACTION_ACCESS_TOKEN }} # 此处 ACTION_ACCESS_TOKEN 与源码仓库添加的 personal_token 命名一致
        external_repository: username/username.github.io # 发布到哪个仓库，此处选 Github Pages 所在仓库
        publish_branch: main # 发布到哪个 branch
        publish_dir: ./public # 发布哪个文件夹的内容，由于 hugo 默认生成静态网页到 public 文件夹，所以这里发布 public 文件夹里的内容

commit 后，回到本地 blog-source-code 文件夹，执行下述 git 命令行来拉取更改至本地：

git pull origin main

此时，源码仓库的 GitHub Actions 已配置完成。当对 blog-source-code 仓库进行 push 操作时，会自动触发上述Action，进而将编译好的页面推送至 username.github.io 仓库，实现博客的自动化编译发布。

评论系统-gitalk

Gitalk 是一个基于Github Issue 的评论插件。

创建仓库，打开issues功能

在 Github 创建一个仓库用于存放评论。如果博客本身使用的就是 Github Pages，此步可省略。

注册 Github Application

依次点击 Github 头像 - Setting - Developer Settings - OAuth Apps - New OAuth Apps

  • Application name: 随便写
  • Homepage URL：博客的网址，https://username.github.io/
  • Authorization callback URL: github权限验证的回调地址，默认是域名，https://username.github.io/

创建成功后获取 Client ID 和 Client Secret 并保存。

在博客中集成gitalk

打开 config.toml 修改相应配置

[params.comment.gitalk]
enable = true
clientId ="xxx"
clientSecret = "xxx"
repo = "username.github.io"
owner = "username"
admin = "username"

推送到远端

git add .
git commit -m "add gitalk"
git push -u origin master

VS code 配置

主要使用两个插件:

  1. Markdown All in one Markdown 写作环境配置
  2. Paste Image 图片插入，参考如下配置
{
    "pasteImage.basePath": "${projectRoot}/static",
    "pasteImage.path": "${projectRoot}/static/images",
    "pasteImage.prefix": "/"
}
  • basePath 图片基础路径，即在markdown文件里面不显示static，只从images开始显示
  • path 图片存放的位置
  • prefix 路径前缀

关于图床

尝试过使用 picgo + GitHub 仓库 + CDN 加速 的方式建立图床，实现图片远程访问，但 CDN(jsdeliver) 加速用了一天就挂了，其他免费的稳定性也都一般。考虑到目前所用图片不多，就先暂时放到默认文件夹 /static/images 和源代码一起推送至 Github 仓库，以后有时间再折腾。
